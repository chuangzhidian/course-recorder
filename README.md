# course-recorder · 在线课程挂机录屏工具

自动化"点下一节 + 播放"，配合 OBS 把整门加密课程（画面 + 声音）录成本地视频。

> **用途边界**：本工具**不下载、不解密**任何视频，只是自动化你手动就能做的操作（切课时、点播放）。
> 仅用于录制**你已购买 / 有权观看**的内容，供个人离线学习。加密视频（如保利威 .pts）无法直接下载，
> 录屏是唯一合规可行的离线方式。

---

## 适用平台

| 脚本 | 平台 | 说明 |
|------|------|------|
| `shenlan_autoplay.user.js` | 深蓝学院 shenlanxueyuan.com | 视频为保利威(Polyv)加密流，用录屏 |

**深蓝播放器的技术特点**（脚本据此设计）：
- 视频在**同源 iframe** `#task-content-iframe` 内；
- 但保利威用 **WASM 解码 + 画到 `<canvas>`**（防下载），页面里的 `<video>` 是空壳、读不到时间；
- 所以脚本改为**读控制条时钟**判断进度与结尾：`.pv-time-current`（当前）/ `.pv-time-duration`（总时长）；
- 开播需点封面中央的大播放键 `span.pv-icon-btn-play`（直接 `video.play()` 无效）。

## 一次性准备

### 1. 装油猴脚本
- 浏览器装 [Tampermonkey](https://www.tampermonkey.net/) 扩展；
- Tampermonkey → 新建脚本 → 把 `shenlan_autoplay.user.js` 内容整段粘进去保存。
- 命令行把脚本复制到剪贴板：`pbcopy < shenlan_autoplay.user.js`

### 2. 装 BlackHole（内录系统声音，免费）
```bash
brew install blackhole-2ch
```
装完若"输出"列表里没有 **BlackHole 2ch**，重载音频服务（或重启电脑）：
```bash
sudo killall coreaudiod
```

### 3. 装 OBS（免费）
```bash
brew install --cask obs
```
**首次打开 OBS 必做**：`系统设置 → 隐私与安全性 → 屏幕录制` 勾选 **OBS**，然后重开 OBS，否则录出来是黑屏。

## 每次录制流程

### 第 1 步：用带 flag 的 Chrome 启动
无人值守时脚本要自动播放**有声**视频，浏览器默认拦截无手势有声自动播放。先 `Cmd+Q` **完全退出 Chrome**，再：
```bash
open -a "Google Chrome" --args --autoplay-policy=no-user-gesture-required
```
用 `chrome://version` 确认"命令行"里有 `--autoplay-policy=no-user-gesture-required`，然后进课程页。

### 第 2 步：设置声音（决定吵不吵）
- **挂机睡觉（静音录制，推荐）**：`系统设置 → 声音 → 输出` 选 **BlackHole 2ch**。
  声音只进虚拟声卡被 OBS 录下，**房间里没声音**。
- **想边录边听**：在"音频MIDI设置"里建"多输出设备"= BlackHole + 扬声器/耳机，输出选它。

### 第 3 步：配 OBS
- **画面**：来源 → `+` → **显示器采集**（整屏，最稳）。
  - 注意：**别用播放器全屏**——脚本要看到左侧目录才能点下一节。保持课程页正常布局。
- **声音**：设置 → 音频 → 桌面音频 → 选 **BlackHole 2ch**；麦克风/辅助音频 → **停用**（不录房间声）。
- **录像格式**：设置 → 输出 → 录像格式选 **mkv**（长时间挂机更安全，中途崩了文件也不废；mp4 崩了会整段损坏）。
- **可选自动分段**：高级输出模式 → 录像 → 勾"自动断录"，按 60 分钟切一个文件，便于管理。
- 帧率 30 即可（讲课/PPT 类足够）。

### 第 4 步：开录 + 启动连播
1. OBS 点 **开始录制**；
2. 回到课程页，点右下角面板 **▶ 从第1节**；
3. 脚本自动：开播 → 播到结尾 → 等 4 秒 → 切下一节 → 再开播 …… 直到全部放完；
4. 睡觉。早上 OBS **停止录制**，系统输出切回扬声器。

### 第 5 步：录完后处理
- **mkv 转 mp4**：OBS 菜单 `文件 → 重新封装录像(Remux)` → 转成 mp4；
- **切成单节**：点面板 **📋 分段表** 复制"课时→开始时间"清单，按时间点在剪辑软件（剪映 / ffmpeg）里切片。

## 控制面板（右下角，可拖动/折叠）

| 按钮 | 作用 |
|------|------|
| ▶ 从第1节 | 清空分段表，从第 1 个视频课时开始自动连播 |
| ▶ 从当前节 | 从你当前打开的这节开始 |
| ⏹ 停止 | 停止自动连播（不影响视频本身播放） |
| ⏭ 手动下一节 | 手动跳到下一节 |
| 📋 分段表 | 复制每节开始时间，供录后切片 |
| ⏮ 每节从头 / ⏯ 续播历史 | 切换：每节从 00:00 录，还是从上次进度续播 |

面板底部小字区显示运行日志（`▶ 第N节开始` / `⏭ 播放到结尾` / `⏮ 检测到续播，拉回从头`）；
控制台里 `[SL-AUTO]` 开头是同样日志。拖**标题栏**移动面板，点右上角 **`—`** 折叠。

## 可调参数（脚本顶部）

| 参数 | 默认 | 说明 |
|------|------|------|
| `BUFFER_MS` | 4000 | 一节结束后等几秒再切下一节（给录屏留干净分段边界） |
| `SAFETY_PAD_MS` | 120s | 单节看守 = 剩余时长 + 该余量，防时钟卡住时卡死 |
| `NO_VIDEO_MS` | 15s | 点开某节 N 秒内没出现时长时钟 → 判为非视频(PDF/课件)，自动跳过 |

## 工作原理（简述）

- 目录里含【视频】的 `li.task-item` 即课时列表，按 DOM 顺序=课程顺序；用 `/task/<id>/` 把当前节锚定到列表下标（防跑偏），整页刷新用 localStorage 续上。
- 每秒轮询控制条时钟：`当前 ≥ 总时长 - 2s` 判为结尾 → 推进下一节。
- 「从头」靠检测时钟**跳进**（一次跳 >20s = 续播/定位）触发拉回 0；正常逐秒播放不触发，故开头不会反复重播。
- 只在**顶层页面**运行（`window.top === window.self`），避免播放器 iframe 里跑出第二份实例互相打架。

## 排障

- **不自动开播**：确认 Chrome flag 生效（`chrome://version`）；播放器改版导致播放键选择器失效时，在控制台看 `[SL-AUTO]`，对 `.pv-icon-btn-play` 等选择器。
- **到结尾不跳 / 反复重播**：确认时钟元素类名还是 `.pv-time-current` / `.pv-time-duration`（保利威改版可能变），变了就更新 `readCur/readDur`。
- **"从头"没生效**：保利威 WASM 播放器 seek 接口不固定，`seekZero()` 里按 `player.seek(0)` 等尝试+兜底点进度条；失效则需重新抓其播放器 JS API。
- **录像没声音**：系统输出、OBS 桌面音频都要选 **BlackHole 2ch**。
- **录出来黑屏**：OBS 没给屏幕录制权限（见"装 OBS"）。
